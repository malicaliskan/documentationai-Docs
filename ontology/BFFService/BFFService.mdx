# BFFService

**MPO Version:** 1.3.0

Configuration settings for the BFF (Backend-for-Frontend) service in the project. Although optional, a BFF service is highly recommended in microservice architectures to consolidate and simplify data access for frontend clients. Since data is distributed across services, the BFF service acts as a unified interface, retrieving and aggregating data from different sources. Mindbricks supports a hybrid CQRS strategy where complex aggregations, defined as DataViews, can be stored and queried efficiently via Elasticsearch. Alternatively, lightweight aggregations can be executed dynamically using dynamic DataViews, which merge data from services at request time without persisting the result. Use Sttaic DataViews for performance-critical or cross-service filtered views, and Dynamic DataViews for dynamic, user-specific aggregations. If no DataViews are defined, the BFF service will not be generated for the project.

```typescript
interface BFFService = {
  dataViews : DataView[];
}
```

| Field     | Description                                                                                                                                                                                                                                                |
| --------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| dataViews | A list of DataView definitions that describe how data should be aggregated and exposed by the BFF service. These views can be optionally persisted in Elasticsearch for efficient querying, or resolved dynamically depending on performance and use case. |
